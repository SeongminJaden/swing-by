# PRD: VidEplace (비드플레이스)

> **"Write the idea. We build, verify, and ship it."**

---

## 1. 서비스 개요

### 서비스명 후보

| 이름 | 의미 | 비고 |
|------|------|------|
| **VidEplace** | Vibe + Dev + Place (바이브코딩이 이루어지는 공간) | 현재 리포명, 직관적 |
| **ShipFlow** | Ship(배포) + Flow(흐름) | PRD→코드→검증→배포 플로우 강조 |
| **LaunchDeck** | Launch(발사) + Deck(조종석) | 배포+모니터링 컨셉 |
| **CodeOrbit** | Code + Orbit(궤도) | "코드를 궤도에 올린다" |
| **ForgeAI** | Forge(대장간) + AI | AI가 만들어내는 이미지 |

### 한 줄 정의

**PRD를 작성하면 AI가 코드를 생성하고, 자동 보안/품질 검증 후 원클릭 출시까지 해주는 올인원 Electron IDE**

> BYOK (Bring Your Own Key) 방식 — 사용자가 자신의 OpenAI, Gemini, Claude 등 API 키/계정을 연결하여 사용

### 핵심 가치

```
"비개발자도 아이디어만 있으면 안전한 프로덕션 서비스를 만들 수 있다"
```

### 타겟 유저

| 세그먼트 | 설명 | 페인포인트 |
|----------|------|-----------|
| 바이브코더 | 프롬프트로 코딩하는 비개발자/초보 개발자 | 만든 코드의 품질·보안을 모름, 출시를 못 함 |
| 1인 창업자 | 빠르게 MVP 만들어야 하는 사람 | 개발+출시+운영 전부 혼자 해야 함 |
| 소규모 팀 | 개발자 1~3명 스타트업 | 인프라/DevOps에 시간 쓰기 아까움 |
| 비개발 부서 | 사내 도구를 직접 만들고 싶은 기획자/마케터 | IT팀 의존 없이 도구를 만들고 싶음 |

### 모듈 B-1: AI 프로바이더 연동 (BYOK)

> OpenCoder, Oh My Open Code처럼 **사용자 소유의 AI 계정/키를 연결**하는 구조.
> 우리 서버를 거치지 않고 사용자 → AI 프로바이더 직접 통신 (비용 = 사용자 부담).

**지원 프로바이더 & 연결 방식:**

> 핵심 원칙: **API 키를 모르는 사용자도 "로그인만 하면" 바로 AI 사용 가능**
> 고급 사용자는 API 키 직접 입력도 가능 (옵션)

| 프로바이더 | 1순위: 로그인 연결 | 2순위: API 키 (고급) | 지원 모델 |
|-----------|-------------------|---------------------|----------|
| **Anthropic (Claude)** | Claude.ai OAuth 로그인 | API Key 입력 | Opus, Sonnet, Haiku |
| **OpenAI** | ChatGPT OAuth 로그인 | API Key 입력 | GPT-4o, GPT-4.1, o3 |
| **Google (Gemini)** | Google 계정 로그인 | API Key 입력 | Gemini 2.5 Pro, Flash |
| **Ollama (로컬)** | 로컬 자동 감지 | 서버 URL 수동 입력 | Llama, Mistral 등 |
| **OpenRouter** | — | API Key 입력 | 다양한 모델 통합 |

**AI 연결 핵심 설계:**

| 항목 | 설계 |
|------|------|
| 연결 방식 (기본) | **OAuth 로그인** — 사용자가 claude.ai/chatgpt.com/google 계정으로 로그인하면 자동 연결 |
| 연결 방식 (고급) | API 키 직접 입력 — 개발자/파워유저용 옵션 |
| 인증 토큰 저장 | Electron keytar (OS 키체인) — 로컬 암호화 저장, 서버 전송 X |
| 구독 감지 | 로그인 시 Pro/Plus/Free 구독 상태 자동 감지 → 사용 가능 모델 표시 |
| 통신 경로 | Electron Main Process → AI 프로바이더 직접 호출 |
| 비용 투명성 | 각 요청별 토큰 사용량 + 예상 비용 실시간 표시 |
| 모델 라우팅 | 작업 유형별 다른 모델 지정 가능 (비용 최적화) |
| 폴백 | 1순위 모델 실패 시 → 2순위 모델 자동 전환 |
| 사용량 표시 | 구독 한도 대비 사용량 게이지 표시 |

| **Network** | HTTP 요청/응답 모니터링, 상태코드, 응답시간, 페이로드 |
| **Problems** | ESLint 경고, TypeScript 에러, 보안 스캔 결과 통합 표시 |
| **Terminal** | 내장 터미널 (npm run, git 등 직접 명령 실행) |

**디버그 → AI 연동:**
- 에러 로그 옆 `[AI에게 수정 요청]` 버튼 → 에러 컨텍스트가 AI 채팅으로 자동 전달
- AI가 에러 원인 분석 + 수정 코드 제안
- Network 탭의 500 에러 → AI가 서버 코드 추적하여 원인 파악

**AI 코드 생성 플로우:**
```
1. 사용자가 AI 채팅(챗 형식)에서 PRD 또는 자연어로 요구사항 입력
2. AI가 서비스 구조 제안 → 사용자가 채팅에서 승인
3. 파일 단위로 코드 생성 (채팅에서 실시간 진행률 표시)
4. 미리보기 패널에서 결과 즉시 확인
5. 생성된 코드 자동 보안/품질 검증 → 디버그 콘솔 Problems 탭에 표시
6. 검증 실패 시 → AI가 자동 수정 제안 (채팅에서 승인/거절)
7. 디버그 콘솔의 에러 → 원클릭으로 AI에게 전달 → 자동 수정
```

**모니터링 에이전트 삽입 방식:**
```
출시 시 자동으로 경량 에이전트 코드 삽입:

Next.js → middleware.ts에 메트릭 수집 코드 추가
Express → 미들웨어로 자동 삽입
Flask → WSGI 미들웨어로 자동 삽입

수집 데이터:
- HTTP 요청/응답 (상태코드, 응답시간, 엔드포인트)
- 에러/예외 (스택트레이스 포함)
- console.log/warn/error
- 서버 리소스 메트릭 (CPU, 메모리)
- DB 쿼리 (실행시간, 쿼리 내용)

→ WebSocket으로 VidEplace 앱에 실시간 전송
→ 로컬 저장 (SQLite) + 필요 시 클라우드 백업
```

**핵심 차별점:**
1. **보안 검증 게이트** — 검증 없이는 출시 불가 (안전장치)
2. **로컬 파일 접근** — 웹 IDE의 한계를 Electron으로 극복
3. **풀 라이프사이클** — 생성부터 운영까지 한 곳에서
4. **AI 네이티브** — 모든 단계에 AI가 관여 (생성, 검증, 모니터링 추천)
5. **IDE를 떠나지 않는 UX** — 모든 로그인/설정이 앱 내부에서 완결
6. **BYOK (자기 AI 계정)** — 로그인만 하면 연결, AI 비용 사용자 부담으로 플랫폼 가격 저렴
